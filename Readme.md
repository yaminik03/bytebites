## Reflection:

The core concept of this project was understanding how different classes interact in a small system, such as how MenuItem, Menu, Order, and User work together to simulate a simple ordering system. Students needed to understand object-oriented programming concepts like class design, method behavior, and how data flows between objects. A common challenge was keeping the order of parameters consistent between the class constructor and the tests, which caused errors when the wrong data type was passed. AI tools were helpful for generating test ideas and explaining error messages, but they sometimes produced code that did not match the existing class definitions, which required careful review. Students also needed to learn how to interpret test failures and trace them back to the underlying code. If guiding a student, I would ask them to print or inspect the values being passed into the constructor and compare them with the class definition to help them discover the issue themselves rather than giving the direct fix.
